# プロジェクトルール

## GAS デプロイ手順（必須）

GAS（Google Apps Script）のコードを変更した場合、以下を必ず守ること。

### 1. コミット・マージを先にやる
コード変更後、GASにデプロイする**前に**必ず:
- git commit する
- main ブランチにマージする

### 2. デプロイは main の最新コードから行う
`clasp push` は全ファイルを丸ごと上書きするため、古いブランチからデプロイすると他の変更が消える。
**必ず main ブランチの最新状態からデプロイすること。**

### 3. 全デプロイメントを更新する
`gas_deploy.sh` を使って全デプロイメントを一括更新する:
```
bash gas_deploy.sh <GASソースファイルのディレクトリ>
```

### 手順まとめ
```
git add <変更ファイル>
git commit -m "..."
git checkout main && git merge <ブランチ>   # or worktreeならmainにマージ
bash gas_deploy.sh <mainのGASファイルがあるディレクトリ>
```

## 既存機能を壊さない

- 新機能追加時、既存のコマンドやフローが動作し続けることを確認する
- GASにデプロイする際は、全ファイルが含まれていることを確認する（`gas_deploy.sh` の出力でファイル数をチェック）
